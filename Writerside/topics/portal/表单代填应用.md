# 表单代填

## 参数说明

| 参数             | 是否必选 | 说明                                                         |
| :--------------- | :------: | ------------------------------------------------------------ |
| 登录提交 URL     |    是    | 登录表单提交完整 URL，以 http:// 或 https:// 开头，如：https://oa.xxxx.com/login |
| 登录名属性名称   |    是    | 表单中用户名标签的 name 属性                                 |
| 登录密码属性名称 |    是    | 表单中密码框标签的 name 属性                                 |
| 登录密码加密算法 |    否    | 登录密码加密算法，目前支持 AES 和 BASE64                     |
| 登录密码加密秘钥 |    否    | 登录密码加密秘钥，当选择 AES 加密算法时显示                  |
| 用户名加密算法   |    否    | 用户名加密算法，目前支持 AES 和 BASE64                       |
| 用户名加密秘钥   |    否    | 用户名加密秘钥，当选择 AES 加密算法时显示                    |
| 登录提交方式     |    是    | 表单登录提交方式                                             |
| 授权范围         |    是    | 规范应用的可使用人群范围 ，**全员访问：** 均可访问该应用，无需额外授权。 **手动授权：**需要在应用的**【访问授权】**标签中，手动分配可访问应用的组织和账户。 |
| 登录其他字段     |    否    | 登录表单中的其他字段或属性                                   |

## 常见问题

### 前后端分离的应用是否支持表单代填？

分情况，表单代填可直接请求应用登录接口，如果返回为token，那么定然是无法支持的，需要进行小改造，如果客户端直接重定向到登录页面，那是支持的。

### 有图片验证码支持表单代填吗？

不支持，Csrf 应用也不支持表单代填

### 如何获取登录的请求连接和登录参数？

在SP登录页面，通过F12打开 network，获取到登录请求的 url。 并且获取到登录的参数，添加到表单代填应用配置中。

### 应用不支持表单代填，如何进行单点登录对接？

如果是支持标准协议的，可以使用SAML、OIDC等协议对接； 如果是自建应用，支持改造的，又不想太麻烦可以使用JWT进行单点登录。
